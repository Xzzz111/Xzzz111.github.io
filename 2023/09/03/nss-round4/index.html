phar文件相关知识

一个提交文件和一个查询文件。
先查查upload.php和index.php是什么。
upload.php:

1、upload.php对上传文件类型做了白名单限制，而且还对文件内容是否有__HALT_COMPILER();?>做了判断，这是phar文件的特殊属性。
2、php识别phar文件是通过其文件头的stub，更确切一点来说是__HALT_COMPILER();?>这段代码，对前面的内容或者后缀名是没有要求的，可以直接修改为其他后缀。
3、将phar文件使用 gzip 命令进行压缩，可以看到压缩之后的文件中就没有了__HALT_COMPILER()
index.php:

看到这个class就知道是反序列化了，我们需要利用file_get_contents()来执行恶意代码。
4、phar文件之所以能反序列化，是因为php使用phar_parse_metadata在解析meta数据时，会调用php_var_unserialize进行反序列化操作。
5、已知反序列化会调用wakeup，根据index.php,我们知道要绕过wakeup,__wakeup失效这个漏洞的原因是序列化后的字符串属性个数不正确时，process_nested_data函数返回0导致call_user_function_ex函数不执行从而不调用__wakeup()。
6、注意：winhex打开phar文件修改序列化字符串的成员个数，由于phar文件带有签名校验，需要把签名部分也改了，否则会报错。

CVE-2021-41773
这是一个存在于Apache HTTP Server 2.4.49 上的漏洞
poc:
GET /icons/.%%32%65/.%%32%65/.%%32%65/.%%32%65/.%%32%65/etc/passwd HTTP/1.1 (实测/.%2e/%2e.....也可以)
exp:

漏洞利用就是这么一回事，原理我也不懂。
这里对URL的处理有点看不懂。询问后得知，中间件和php解析器可能会处理/..和/.还有/，这样的话原本的../../../../就得通过把.编码成%2e来绕过，考虑到curl或者burp请求的时候可能会自动url解码一次，那么就可以把2e编码成%32%65，或者curl的时候加上–path-as-is选项。
这个东西很玄学，反正注意burpsuite里面右键有一个选项是URL编码输入，不小心开了之后会自动URL编码输入。
这个回答会有点启发：https://github.com/httpie/cli/issues/895
