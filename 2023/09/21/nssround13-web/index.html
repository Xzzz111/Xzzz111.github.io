基础知识：
JWT 与 Token 介绍
理解Cookie和Session机制
对flask session伪造的学习

我的理解是这样的：cookie是一段数据，可以用来装sessionid或者token。session和token是用来‘鉴权’的机制，jwt是一种token机制的实现方法。
session是服务器开辟一块空间来存储用户的session信息，同时返回sessionid到客户端。
token是把用户信息直接存到客户端，通过某种加密来使得它无法被破译。服务器只需要保存密钥就可以了。

和之前那题差不多，我先注册了一个用户登陆，这个页面的源代码里有一个提示说密钥在/wor。

那么我们直接进到/wor下，这里莫名其妙的显示了上一次登录的时间。很离谱的是我以为这里是要我构造一个请求包，在header里面把时间改为他给我的时间，还查了date请求头，但是好像没什么用。
然后目光转向session，拿去解码，我发现原来这玩意不用密钥就能解出来，有时间可以看看原理。解开后和之前那题一样，只是多了一个time字段，正是web页面显示的时间。
这里应该是session中的时间被读取了，然后显示到前端。我们这里可以随便改一下session，触发报错。

看到密钥了，而且代码印证了我们的想法，读取session中的时间，如果读取不到的话就会return '' %logintime这行代码会引发报错。
拿到密钥后就可以构造session的值了，这里要么把time字段删除了，要么把time字段后面的值加上引号，不然服务器处理这段session可能会出错。

总之，关键是这里，当然前提有三个条件来着（在参考文章里可看），解释是这样的：

  1. RewriteRule "^/nssctf/(.*)" "http://backend-server:8080/index.php?id=$1" [P]：这是一个重写规则，它告诉 Apache 服务器如何处理以 /nssctf/ 开头的 URL。当用户访问匹配此规则的 URL 时，服务器将将请求重写到 "http://backend-server:8080/index.php?id=$1"。其中，$1 是匹配到的 URL 中的任何文本，这部分文本将作为参数传递给后端服务器的 index.php 脚本。"[P]" 表示这是一个代理规则，即 Apache 服务器将代理请求到后端服务器。

  2. ProxyPassReverse "/nssctf/" "http://backend-server:8080/"：这是一个代理反向设置，用于确保在代理请求时正确处理响应。当后端服务器响应时，它会返回一些可能包含重定向或链接的头信息。这个指令告诉 Apache 服务器在将响应返回给客户端时，将任何来自 "/nssctf/"的链接或重定向重写为指向 "http://backend-server:8080/"，以确保客户端正确访问后端服务器。

综合起来，这段配置的作用是将以 "/nssctf/" 开头的 URL 代理到后端服务器的 "http://backend-server:8080/index.php"，同时确保处理响应中的链接和重定向，以使它们指向正确的后端服务器地址。这有助于在 Web 应用程序中实现反向代理和重定向功能。

有两个参数mode和value，当mode为save时，会把/tmp/data.json的内容传给$data。然后把value经过addslashes处理过后赋值给$value，再将$data中的]给替换为, '$value']，然后再把改过的$data，写到/tmp/data.json中。

根据回显可猜测原本可能是{"test":[1]}随着我们传入2后,]被改写为,2]，这样整个json就是{"test":[1,2]}，后面3、4也是同理。同时注意到如果题目这样设置的话，我们也不能随意测试了，不然会因为前面的错误格式导致程序终止而无法读取到后面的代码。

因为我们要执行恶意代码echo 'cat /flag'这样的话就会受到addslashes函数的限制。

addslashes() 函数是 PHP 中用于将字符串中的某些特殊字符进行转义的函数它的作用是在以下字符前面插入一个反斜杠\：

  • 单引号（’）
  • 双引号（”）
  • 反斜杠（\）
  • NUL（NULL字符）

这样的话就要用反引号`来替代引号了，反引号可以用来执行系统命令，相当于shell_exec，但是不会打印结果,system()会自动打印结果。
