chrck(v1)

有点像之前打的ciscn2023里面的unzip

开局先看源码

flask框架搭的一个web服务，包括了用户上传文件/upload和下载文件/download。

仔细看upload_file部分：

  • 上传的文件只能为tar包，而且文件名不能为空且不包含..和/
  • 成功上传后会打开tar包并且解压到./uploads文件夹下。
  • 按道理重复上传同样的文件会报This file already exists但是实际上每次都是success，可能是因为源码在上传完之后有个os.remove(file_save_path)删除了文件。

再看download_file:

  • 需要post一个filename，这个filename要不为空且不能包含..或/
  • 在保存上传文件的目录下查找是否有叫filename的文件
  • 读取这个文件
    我们可以建立一个软链接到/flag下，然后把它打包成tar包，这样上传后解压出来的是我们的软链接，/download里面读取这个软链接会直接读/flag。

    写python脚本上传并下载文件。
    注意软链接不要写相对路径，写绝对路径不容易出错，而且我们不明白服务器端的文件结构。
