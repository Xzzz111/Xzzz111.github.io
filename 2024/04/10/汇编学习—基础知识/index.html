寄存器

寄存器是什么？
寄存器(Register)是中央处理器(CPU，Central Pro
cessing Unit )内部的组成部分,寄存器是CPU中存储器的一部分。
–寄存器是有限存储容量的高速存储部件，它们可用来
暂存指令、数据和地址
–寄存器是计算机系统结构下存储层次的最顶端，也是
系统中操作数据的最快速路径

寄存器的作用是临时存储数据和地址，供CPU操作，包括：

  1. 8个通用寄存器（EAX,EBX,ECX,EDX,EDI,ESI,ESP,EBP）
  2. 1个指令指针寄存器（EIP），EIP始终指向下一条待执行指令的
    地址
  3. 一个CPU状态寄存器（EFLAGS）
  4. 6个段寄存器

下面来介绍上述每个寄存器

  1. –EAX：累加寄存器，是操作数和结果数据的累加器
    –Extended Accumulator Register
  2. –EBX：基址寄存器，指向DS段中数据的指针
    –Extended Base Register
  3. –ECX：计数寄存器，是字符串和循环操作的计数器
    –Extend Count Register
  4. –EDX：数据寄存器， I/O指针
    –Extend Data Register
    (上面4个寄存器主要用于算术运算（ADD/SUB/XOR/OR等）
    指令中，常用来保存常量与变量的值)
  5. –ESI：(字符串操作源指针)源变址寄存器
    – Extended Source Index
  6. –EDI：(字符串操作目的指针)目的变址寄存器
    –Extended Destination Index
    (ESI和EDI与特定的串操作指令（MOVS/LODS/STOS）一起使用，在字符串操作的时候用的比较多)
  7. -ESP：栈顶指针寄存器，用于存放当前堆栈的栈顶地址，专门用作堆栈指针，不可作为一般通用寄存器使用
  8. –EBP：栈底指针寄存器（基址指针寄存器），表示栈区域的基地址，永远指向当前函数栈的栈底位置， 不可作为一般通用寄存器使用
  9. CS：Code Segment，代码段寄存器
    –存放应用程序代码所在段的段基址
  10. DS：Data Segment，数据段寄存器
    –用于存放数据段的段基址
  11. SS：Stack Segment，堆栈段寄存器
    –用于存放栈段的段基址
  12. ES、FS、GS，附加数据段寄存器
    –用于存放程序使用的附加数据段的段基址
  13. EIP：Extended Instruction Pointer，指令指针寄存器
    –在32位系统中：
    它的大小为32位，是由原来的16位IP寄存器扩展而来,直接保存CPU下一条将要执行指令的地址
  14. EFLAGS 大小为4个字节， 共有32个位元，每一位都有专门的含
    义，记录特定的信息，每位的值为1或0，代表On/Off
    或True/False,具体有哪些标志位不细说了。

在x86架构中，由于是32位计算机，因此寄存器也是32位大小，这里的E指的就是Extended，即扩展的。比如eax为32位，ax就是16位，ah就是ax中的高8位，al就是ax中的低八位。如果是x64架构，就会出现如rax这样的寄存器，作用和eax一样，只是大小变成64位了。

基本指令

这里的指令都是intel风格的，即大致为：指令 目的操作数 操作数

  1. MOV DEST SRC
    OPS可以为：存储器、通用寄存器、段寄存器和立即数。
    OPD可以为：存储器、通用寄存器和段寄存器（CS除外）
    注意：
    -立即数不能送段寄存器，其余可以任意搭配；立即数送存储器的指
    令有时难以确定操作数的长度，需要在存储器操作数的前面加上类
    型说明BYTE PTR或WORD PTR。
    例如：
    MOV BYTE PTR[SI+10H]，30 ；8位立即数30送偏移地址为SI+10H的字节单元；
    MOV WORD PTR[BX+DI]，2 ；16位立即数2送偏移地址为BX+DI的字单元
    -两存储单元(内存存储单元，即存储操作数)之间不能直接进行数据传送；例如，MOV [2000H]，[BX];是错误指令。
    两个段寄存器之间不能直接进行传送信息。但可以用CPU内部寄存器为桥梁来完成这样的传送
    – 例如：MOV AL，AREA1
    MOV AREA2，AL
    例如： MOV AX，1000H
    MOV DS， AX
    –立即数、代码段寄存器CS只能作源操作。
    –IP寄存器不能作源操作数或目的操作数。
  2. LEA DEST SRC
    和MOV的主要区别在于它是取源操作数的地址赋给目的操作数，对于寄存器来说:第二个操作数是寄存器必须要加[]。例如lea eax [ebp+i]这样的指令就相当于是把ebp+i的值赋给eax，所以lea指令在某种情况下也能当乘法使用。
  3. PUSH
    入栈，将操作数入栈后再将esp往下挪一次（栈由高到低生长）。
  4. POP
    出栈，先将esp指向的数据赋给操作数，然后再把栈顶指针向上挪回去。
  5. CALL
    调用函数，这个指令要先经历将call的下一条指令地址入栈，然后再跳转到call调用的子函数的地址。
  6. RETN
    call的逆操作，相当于是pop eip(实际上没有这条指令，只是作用相似)。
    除此以外还有许多跳转指令和对数据进行操作的指令等，直接默认能用就行。
